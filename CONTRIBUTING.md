## How Can I Contribute?

### 🐛 Reporting Bugs
- Check if the issue already exists in [GitHub Issues](../../issues)
- If not, open a new issue with a clear title and description
- Include steps to reproduce, expected behavior, and actual behavior

### 💡 Suggesting Features
- Open a [GitHub Discussion](../../discussions) to propose your idea
- Explain the use case and why it would benefit retailers

### 🛠️ Submitting Code
- Fix open issues labeled `good first issue` or `help wanted`
- Improve documentation or tests
- Add support for new accounting/POS system integrations

## Getting Started

1. **Fork** the repository
2. **Clone** your fork:
   ```bash
   git clone https://github.com/YOUR_USERNAME/retailpulse-ai.git
   cd retailpulse-ai
   ```
3. **Install dependencies:**
   ```bash
   npm install
   ```
4. **Copy environment variables:**
   ```bash
   cp .env.example .env
   ```
5. **Create a branch:**
   ```bash
   git checkout -b feature/your-feature-name
   ```

## Pull Request Process

1. Make sure your code runs without errors
2. Update the `README.md` if you've changed any functionality
3. Add or update tests where applicable
4. Submit your Pull Request with a clear description of what you changed and why
5. Wait for a maintainer to review — we aim to respond within 3 business days

---

## Commit Message Guidelines

Use clear, descriptive commit messages:

```
feat: add inventory deadstock detection
fix: resolve API timeout on large datasets
docs: update README with setup instructions
refactor: simplify local agent data pipeline
```

**Prefixes:**
- `feat` — new feature
- `fix` — bug fix
- `docs` — documentation only
- `refactor` — code restructure, no feature change
- `test` — adding or updating tests
- `chore` — maintenance tasks
